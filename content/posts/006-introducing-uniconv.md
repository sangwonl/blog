### 온라인 컨버터, 뭔가 찝찝하다

HEIC 사진을 JPG로 바꾸고 싶어서 "heic to jpg"를 검색한다. 첫 번째 결과를 클릭하면 광고가 덕지덕지. 파일을 업로드하려는데 문득 생각이 든다. "이 사진 누가 보는 거 아니야?" 그래도 일단 올린다. 변환이 끝났다. 다운로드 버튼을 누르려는데 또 광고. 겨우 받았더니 이번엔 워터마크가 박혀있다. 프리미엄 결제하라고.

PDF를 DOCX로 바꾸고 싶다. 또 검색. 다른 사이트. 또 광고. 또 업로드. 이번엔 용량 제한에 걸린다.

영상에서 GIF를 뽑고 싶다. 또 다른 사이트. 회원가입하라고 한다.

**매번 다른 사이트, 매번 광고, 매번 개인정보 걱정.** 원하는 변환을 한 곳에서 다 할 수도 없다.

그래서 만들었다. **uniconv**.

### 이름의 의미

uniconv = **Uni**versal **Conv**erter.

세상 모든 게 결국 변환이다. 이미지 포맷 변환, 문서 변환, 영상 인코딩, 텍스트 추출, 심지어 클라우드 업로드도 "로컬 파일 → 원격 저장소"로의 변환이라고 볼 수 있다.

**세상 모든 걸 변환할 수 있는 도구.** 그게 uniconv가 지향하는 방향이다.

### 핵심 아이디어

#### 1. Unix 파이프라인

쉘에서 `cat file | grep pattern | wc -l` 처럼 명령을 연결하듯, 변환도 체이닝할 수 있다.

```bash
# 이미지 → PDF → DOCX
uniconv photo.jpg "pdf | docx"

# HEIC → JPG → 그레이스케일
uniconv photo.heic "jpg | grayscale"

# 하나의 입력으로 여러 포맷 동시 생성
uniconv photo.heic "tee | jpg, png, webp"
```

한 번의 변환으로 끝나지 않는 작업도, 파이프라인 하나로 끝난다.

#### 2. CLI 기반

GUI가 없다. 터미널에서 돌아간다. 왜?

일단 나는 개발자라 CLI가 익숙하다. 그리고 CLI는 **자동화에 유리하다**. 스크립트에 넣을 수 있고, CI/CD에 붙일 수 있다.

거기에 `--json` 옵션을 붙이면:

```bash
$ uniconv --json photo.heic "jpg"
```

```json
{
  "success": true,
  "final_outputs": ["photo.jpg"]
}
```

다른 시스템이 파싱하기 쉽다. LLM 에이전트가 도구로 쓰기에도 좋다.

#### 3. 플러그인 시스템

세상의 모든 변환을 내가 다 개발할 수는 없다. 그래서 **플러그인 시스템**을 도입했다.

```bash
# 이미지 변환 플러그인 설치
uniconv plugin install image-convert

# 문서 변환 플러그인 설치
uniconv plugin install doc-convert

# 기본 플러그인 한 번에 설치
uniconv plugin install +essentials
```

플러그인은 C++로 만들어도 되고, Python이나 Go로 만들어도 된다. 언어에 상관없이 Argument 프로토콜만 맞추면 된다. 커뮤니티에서 누구나 플러그인을 만들어 공유할 수 있다.

#### 4. 무엇이든 될 수 있는 유연함

플러그인이 할 수 있는 일에는 제한이 없다.

- 로컬 파일 포맷 변환 (기본)
- 외부 API 호출 (클라우드 업로드, 외부 서비스 연동)
- AI 모델 실행 (배경 제거, 객체 인식, 텍스트 추출)

예를 들어, 상상해보자면 이런 파이프라인도 가능하다:

```bash
# 이미지에서 얼굴 추출 → S3 업로드
uniconv photo.jpg "faces | s3 --bucket my-bucket"

# 영상에서 하이라이트 추출 → GIF 변환 → 슬랙 전송
uniconv video.mp4 "highlights | gif | slack --channel #random"
```

"변환"의 정의를 넓게 잡으면, 결국 **입력을 받아서 뭔가 다른 출력을 만드는 모든 것**이 변환이다.

### 예제: 이런 게 된다

```bash
# HEIC를 JPG로 (가장 기본)
uniconv photo.heic "jpg --quality 85"

# 영상을 GIF로 (리사이즈 + 프레임레이트 조절)
uniconv video.mp4 "gif --width 320 --fps 15"

# 여러 포맷으로 한 번에 내보내기
uniconv photo.heic "tee | jpg --quality 90, webp --quality 80, png"

# docx 를 이미지로 변환
uniconv page.docx "pdf | png"

# 폴더 감시하면서 자동 변환
uniconv watch -o ./output/ ./incoming "jpg"
```

### 마치며

온라인 컨버터의 광고, 개인정보 우려, 여기저기 흩어진 기능들이 불편했다. 그래서 로컬에서, 터미널에서, 원하는 변환을 자유롭게 조합할 수 있는 도구를 만들었다.

아직 갈 길이 멀다. 더 많은 포맷, 더 많은 플러그인, AI 기반 추출 기능 등 하고 싶은 게 많다.

**GitHub**: [https://github.com/uniconv/uniconv](https://github.com/uniconv/uniconv)

혹시 "이런 변환이 있으면 좋겠다" 싶은 게 있다면, 플러그인으로 만들어서 기여해주시면 감사하겠다. [플러그인 개발 가이드](https://github.com/uniconv/registry/blob/main/plugins/DEVELOPMENT.md)도 레포에 있으니 참고하시길.

```bash
# 설치
curl -fsSL https://raw.githubusercontent.com/uniconv/uniconv/main/install.sh | bash

# 필수 변환 플러그인 설치 (image, video, doc)
uniconv plugin install +essentials

# 도움말
uniconv --help
```
